## 進化 #1：統一素材來源為 Google Drive

### 問題背景

Joey AI Agent 在建站時需要下載客戶提供的素材（Logo、產品照、參考圖等）。原本使用 GitHub 素材庫（`joey-agent-assets`）存放素材，但在實際操作中遇到問題：

- GitHub 適合存放程式碼，不適合管理大型圖片檔案
- 客戶不熟悉 GitHub，無法自行上傳素材
- 檔案大小限制不利於高解析度圖片

### 解決方案

將所有素材來源統一切換到 Google Drive 公開資料夾，使用 Python 工具 `gdown` 下載。

### 具體改動

#### 1. 更新 CLAUDE.md 系統指南

將「GitHub 素材庫使用方式」整段替換為「素材下載方式（統一使用 Google Drive）」：

- 移除所有 `gh repo clone JoeyVIP/joey-agent-assets` 相關指示
- 新增 `gdown` 下載指令範例
- 記錄成功條件（gdown v5.2.1、不需 OAuth、連結設公開）
- 記錄成功案例（space-cat-cafe-v5 成功下載 3 個檔案）

#### 2. 更新官網專案資料準備指南

修改 `docs/官網專案資料準備指南.md` 中的 Step 4 到 Step 6：

- Step 4：資料夾結構從 GitHub repo 改為 Google Drive 資料夾
- Step 5：上傳流程從 Git 指令改為 Google Drive 網頁上傳 + 設定公開權限
- Step 6：啟動建站的指令從提供 GitHub 連結改為提供 Google Drive 連結

### 修改檔案清單

| 檔案 | 修改類型 |
|------|---------|
| `CLAUDE.md` | 修改 - 素材來源區塊重寫 |
| `docs/官網專案資料準備指南.md` | 修改 - Step 4-6 流程更新 |

### 驗證結果

- 已透過 space-cat-cafe-v5 專案驗證 Google Drive 下載流程
- 成功下載 3 個素材檔案（catcoffeelogo.png 1MB、catcoffeemainv.png 8.5MB、catcoffee店內裝潢.png 2.8MB）
- 全程不需要 OAuth 認證，操作簡單

### 效益

- 客戶只需將檔案丟進 Google Drive 資料夾，設為公開即可
- 支援大檔案，不受 GitHub 的 100MB 限制
- 操作門檻大幅降低，非技術人員也能操作

## 進化 #2：新增前端設計模板系統

### 問題背景

Joey AI Agent 建立網站時，前端設計品質不穩定。主要問題：

1. **AI Slop 美學**：生成的網站經常使用 Inter、Roboto 等 generic 字體，配色千篇一律（紫色漸層 + 白底），一看就是 AI 生成的
2. **風格不一致**：每次建站從零開始，沒有設計參考，成品風格差異大
3. **缺乏行業針對性**：代工廠網站和咖啡廳網站長得差不多，沒有行業特色

### 研究過程

在實作前進行了方案研究，評估了 4 種可行方案：

| 方案 | 說明 | 評估 |
|------|------|------|
| A: Claude 官方 Skill | GitHub 上的 frontend-design Skill | 輕量、官方維護、通用 |
| B: 自建行業模板庫 | 為每個行業寫專屬設計模板 | 客製化高、長期最佳 |
| C: 外部模板整合 | shadcn/ui、Tailwind Plus 等 | 現成品質高、需付費 |
| D: v0.dev 預生成 | 用 AI 工具預生成元件 | 品質極高、有額度限制 |

最終選擇混合方案：A + B 為主，C 作為未來擴展。

### 解決方案

建立一套前端設計模板系統，包含官方設計 Skill 和 4 個行業專屬模板。

### 具體改動

#### 1. 下載 Claude 官方 frontend-design Skill

從 GitHub 下載 Claude Code 官方的前端設計 Skill 到 `skills/frontend-design/SKILL.md`。這個 Skill 的核心理念：

- 設計前先分析：目的、受眾、調性、差異化
- 選擇大膽的美學方向（Brutalist、Maximalist、Luxury、Retro 等）
- 禁止 generic 字體（Inter、Roboto、Arial）
- 禁止 AI slop 配色（紫色漸層 + 白底）
- 每個設計都應該是獨特的，不重複

#### 2. 建立共用設計原則

建立 `skills/templates/base-guidelines.md`，包含所有行業共用的規範：

- 中英文字體配對策略（標題裝飾字 + Noto Sans TC）
- 配色比例原則（60-30-10 規則）
- 排版規範（負空間、層次結構、行高）
- 動畫原則（fadeInUp、Staggered Reveal）
- 背景處理技巧（漸層疊加、紋理）
- 按鈕設計（最小觸控區域 44x44px）
- CSS Variables 必須使用

#### 3. 建立 4 個行業模板

每個模板都包含：美學方向、字體推薦（Google Fonts）、配色方案（3-4 套 CSS Variables）、頁面結構、CSS 範例、參考網站。

**製造業模板**（`manufacturing.md`）：
- 3 種美學風格：現代工業、科技精密、經典專業
- 3 套配色：藍灰專業、工業橘、黑金高端
- 字體：Oswald、Rajdhani、Bebas Neue
- 特色元素：數字統計區（計數動畫）、認證標章牆（灰階 hover 變彩色）
- 參考：Foxconn、Flex、Jabil

**餐廳模板**（`restaurant.md`）：
- 4 種美學風格：文青咖啡廳、高級餐廳、活力餐廳、日式極簡
- 4 套配色：Warm Earth、Elegant Dark、Appetizing、Zen
- 字體：Cormorant Garamond、Playfair Display、Poppins、Zen Kaku Gothic New
- 特色元素：全幅美食 Hero、菜品卡片、營業時間區塊
- 參考：Blue Bottle、Noma、Shake Shack、% Arabica

**品牌官網模板**（`brand.md`）：
- 4 種美學風格：極簡高端、潮流街頭、自然有機、復古懷舊
- 4 套配色：Monochrome、Bold Contrast、Earth Tones、Vintage Warm
- 特色元素：產品 Hero 展示、左右交錯版面、品牌宣言區
- 參考：Apple、Supreme、Patagonia、Diptyque

**企業官網模板**（`corporate.md`）：
- 4 種美學風格：科技現代、專業穩重、創意活力、顧問高端
- 4 套配色：Tech Blue、Corporate Blue、Creative Bold、Premium Dark
- 字體：Plus Jakarta Sans、IBM Plex Sans、Syne、Cormorant Garamond
- 特色元素：漸層 Hero、服務卡片、客戶 Logo 牆、客戶見證
- 參考：Stripe、Deloitte、IDEO、McKinsey

#### 4. 修改 claude_code_service.py

2. 根據行業類型選擇對應模板
3. 完成後執行設計檢查清單（字體、配色、排版、動畫、手機版）

## 技術決策記錄

### 進化 #1 決策

| 決策點 | 選擇 | 原因 |
|--------|------|------|
| 素材存放位置 | Google Drive | 客戶操作門檻低、支援大檔案 |
| 下載工具 | gdown | Mac mini 已安裝、免認證、穩定 |
| 棄用 GitHub 素材庫 | 是 | 不適合非技術人員、大檔案受限 |

### 進化 #2 決策

| 決策點 | 選擇 | 原因 |
|--------|------|------|
| 設計 Skill | Claude 官方 frontend-design | 官方維護、輕量、針對 AI slop |
| 模板格式 | Markdown + CSS 範例 | Agent 可直接讀取、易於維護 |
| 字體來源 | Google Fonts | 免費、CDN 穩定、選擇多 |
| 是否用 shadcn/ui | 暫不採用 | 目前建站以純 HTML/CSS/JS 為主 |
| 是否用 v0.dev | 暫不採用 | 需手動操作、有額度限制 |
