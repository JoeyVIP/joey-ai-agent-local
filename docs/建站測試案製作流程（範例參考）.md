# 建站測試案製作流程 — 範例參考

> 本文件記錄「巨領企業冷凍倉儲官網」測試案的完整製作流程，作為未來真實案例的操作範本。
> 該案網站表現評分 **80+ 分**，流程驗證成功。

---

## 一、整體流程概覽

```
Step 1: 蒐集真實資料（官網爬取 + 網路搜尋）
    ↓
Step 2: 建立資料夾結構
    ↓
Step 3: 下載圖片素材（從官網擷取）
    ↓
Step 4: 撰寫 content.md（依照指南格式）
    ↓
Step 5: 撰寫 README.md（標註真實 vs 生成）
    ↓
Step 6: 上傳素材到 Google Drive（設為公開）
    ↓
Step 7: 發送 LINE 訊息給 Agent 建站
```

---

## 二、Step 1：蒐集真實資料

### 需要蒐集的項目清單

| 類別 | 必要項目 | 備註 |
|------|---------|------|
| **基本資訊** | 公司全名、英文名、成立年份、創辦人 | |
| **聯絡方式** | 電話、傳真、Email、LINE、地址 | 全部要真實 |
| **業務聯繫** | 負責人姓名 + 手機 | 問客戶要 |
| **營業時間** | 週幾到週幾、幾點到幾點 | |
| **核心業務** | 主要服務/產品列表 | |
| **設備/技術** | 使用什麼設備、品牌、規格 | 這是差異化賣點 |
| **知名客戶** | 合作過的品牌 | 增加信任感 |
| **經營理念** | 公司價值觀、使命 | |
| **現有官網** | URL | 用來擷取圖片和對照資料 |

### 蒐集方法

1. **現有官網**：最主要的資料來源，用 WebFetch 抓取頁面內容
2. **Google 搜尋**：補充公司登記資料、新聞報導
3. **客戶訪談**：真實案例時由業務提供（測試案則模擬）

### 本次範例蒐集結果

巨領企業的資料主要從 `https://www.jiuhliing.com.tw/` 擷取，包含：
- 公司簡介、設備介紹、服務項目、經營理念
- 一廠 / 二廠的設備規格（德國 Bitzer、Guntner，義大利 ICAM）
- 聯絡資訊、業務人員手機

## 三、Step 2：建立資料夾結構

```bash
mkdir -p "docs/測試案/{專案名稱}/images"
```

### 命名規範

- 資料夾名稱用英文小寫 + 連字號：`jiuhliing-cold-storage`
- 格式：`{公司英文簡稱}-{行業關鍵字}`

### 標準結構

```
{專案名稱}/
├── content.md      # 結構化建站內容（核心檔案）
├── README.md       # 專案說明、資料來源標註
└── images/         # 圖片素材
```

## 四、Step 3：下載圖片素材

### 圖片分類（建議至少準備這些）

| 類別 | 數量建議 | 檔名規範 | 範例 |
|------|---------|---------|------|
| Logo | 1 張 | `logo.png` | logo.png |
| Hero 橫幅 | 1 張 | `hero-banner.jpg` | hero-banner.jpg |
| 公司/廠房外觀 | 1-3 張 | `company-*.jpg` | company-profile.jpg |
| 設備/產品 | 3-8 張 | `equipment-*.jpg` | equipment-bitzer-compressor.jpg |
| 圖示/Icon | 視需求 | `icon-*.png` | icon-integrity.png |

### 關鍵技術筆記：下載需要 Referer Header

> **重要教訓**：很多官網的靜態資源伺服器會擋沒有 Referer 的請求，直接 curl 會拿到 404。

```bash
# 錯誤做法（會得到 404 HTML）
curl -sL -o logo.png "https://static.iyp.tw/42568/system/logo_Jvwy5.png"

# 正確做法（加上 Referer 和 User-Agent）
curl -sL \
  -H "Referer: https://www.jiuhliing.com.tw/" \
  -H "User-Agent: Mozilla/5.0" \
  -o logo.png \
  "https://static.iyp.tw/42568/system/logo_Jvwy5.png"
```

### 下載後驗證

每次下載完都要驗證檔案是不是真的圖片：

```bash
# 檢查檔案類型和大小
file images/*.jpg images/*.png
ls -lh images/
```

- 正常圖片：`JPEG image data` 或 `PNG image data`，大小 > 5KB
- 失敗下載：`HTML document text`，大小 ~150 bytes（是 404 頁面）

### 本次範例下載結果

共 16 張圖片，全部驗證通過：
- 最小：icon-integrity.png（8 KB）
- 最大：equipment-bitzer-compressor.jpg（1.3 MB）

## 五、Step 4：撰寫 content.md

### 格式依據

遵循 `docs/官網專案資料準備指南.md` 的 Step 3 模板。

### 必要區塊

```markdown
# {品牌名} 官網內容

## 基本資訊          ← 全部用真實資料
## 首頁
### Hero 區塊        ← 主標語 + 副標語 + CTA（可潤飾）
### 數字統計區塊      ← 用真實數據，呈現方式可設計
### 特色區塊          ← 3-4 個核心優勢
## 關於我們
### 品牌故事          ← 基於真實背景擴寫
### 經營理念          ← 真實資料
## 產品/服務介紹      ← 真實服務 + 補充適用對象
## 聯絡資訊          ← 全部真實
## FAQ               ← 基於行業常識生成，6-8 題
## SEO               ← Title / Description / Keywords
## 網站設計需求       ← 模擬業主的風格偏好（重要！）
```

### 「網站設計需求」區塊的撰寫要點

這個區塊是讓 Agent 建站時參考的視覺方向，需要包含：

1. **整體風格**：明確說「要什麼」和「不要什麼」
2. **配色偏好**：給出具體色調方向
3. **重要視覺元素**：哪些內容要突出
4. **頁面需求**：每個頁面的核心目的
5. **參考網站**：有的話附上

### 不同行業的風格方向參考

| 行業 | 風格關鍵字 | 配色方向 | 避免 |
|------|-----------|---------|------|
| 冷鏈/倉儲 | 專業工業風、規模感 | 深藍、鋼灰、白 | 花俏、可愛 |
| 餐廳/咖啡 | 溫暖、有氛圍感 | 大地色、暖色系 | 太冷、太企業 |
| 製造/代工 | 精密、可靠、技術感 | 深色系 + 金屬色 | 太活潑 |
| 品牌官網 | 質感、獨特性 | 依品牌 VI | generic |

### content.md 中圖片的引用方式

```markdown
- **背景圖**：images/hero-banner.jpg
- **圖片**：
  - images/equipment-bitzer-compressor.jpg（德國 Bitzer 冷凍機組）
  - images/equipment-guntner-evaporator.jpg（德國 Guntner 蒸發器）
```

用相對路徑 `images/檔名`，並在括號內加描述，讓 Agent 知道圖片內容。

## 六、Step 5：撰寫 README.md

### 必要內容

1. **用途說明**：這是什麼、用來做什麼
2. **公司簡介**：一段話介紹
3. **資料來源表格**：標註每項內容是「真實」還是「生成」
4. **圖片清單**：列出所有圖片和用途
5. **對應模板**：建站時該用哪個模板
6. **檔案結構**：完整的 tree 圖

---

## 七、Step 6：上傳到 Google Drive

### 上傳內容

將整個資料夾的內容上傳到 Google Drive 一個資料夾中（**扁平結構，不建子資料夾**）：

```
Google Drive/{公司名}-官網素材/
├── content.md              ← 建站內容
├── logo.png
├── hero-banner.jpg
├── company-profile.jpg
├── ... (所有圖片)
```

### 必要設定

- 資料夾權限設為「**任何人都可以查看**」
- 複製分享連結備用

---

## 八、Step 7：發送 LINE 訊息給 Agent

### 訊息模板

```
請幫{公司名稱}建立官方網站。

## 素材來源
Google Drive：{GOOGLE_DRIVE_FOLDER_URL}

請使用 gdown 下載全部素材：
gdown --folder "{GOOGLE_DRIVE_FOLDER_URL}" -O ./assets

## 建站指示

1. 下載素材後，先讀取 assets/content.md，裡面有完整的結構化建站內容
2. 這是{行業描述}，請套用{對應模板名稱}模板
3. 圖片都在 assets 資料夾中，content.md 裡有標註每張圖對應的位置

## 重點提醒

- 風格：{一句話描述風格}
- {2-3 個最重要的設計要求}
- content.md 最後有「網站設計需求」區塊，請仔細參考
```

### 模板對應表

| 行業 | 模板檔案 |
|------|---------|
| 製造業/工業服務 | `skills/templates/manufacturing.md` |
| 餐廳/咖啡廳 | `skills/templates/restaurant.md` |
| 品牌官網 | `skills/templates/brand.md` |
| 企業官網 | `skills/templates/corporate.md` |

## 九、成功案例記錄

### 巨領企業冷凍倉儲官網（2026-02-06）

| 項目 | 內容 |
|------|------|
| 公司 | 巨領企業有限公司 |
| 行業 | 冷凍冷藏倉儲（B2B） |
| 使用模板 | manufacturing.md |
| 圖片數量 | 16 張（官網擷取） |
| 網站評分 | **80+ 分** |
| 資料夾 | `docs/測試案/jiuhliing-cold-storage/` |
| 特殊處理 | 圖片下載需加 Referer header |

### 成功因素

1. **content.md 結構完整**：所有區塊都填寫到位，Agent 不用猜
2. **圖片對應清楚**：每張圖在 content.md 裡都標註了位置和描述
3. **設計需求明確**：在 content.md 最後加了「網站設計需求」區塊，告訴 Agent 風格方向
4. **真實資料 + 合理生成**：基本資訊用真實資料，FAQ 和品牌故事基於真實背景生成

### 可改進方向（下次注意）

- 可多準備一些情境照片（人員作業、貨品進出庫場景）
- 可加入影片素材連結（如果客戶有的話）
- FAQ 可以加入更多價格相關問題（客戶最常問的）
- 如果有客戶的品牌 VI 手冊，配色和字體可以更精準
