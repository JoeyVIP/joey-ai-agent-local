# 官網專案資料準備指南

本指南協助業務團隊將客戶訪談內容轉化為 AI Agent 可用的結構化資料。

---

## 整體流程

```
客戶訪談 → 錄音轉逐字稿 → AI 處理成結構化內容 → 上傳 GitHub → Agent 自動建站
```

只要按照本指南準備好資料並上傳 GitHub，後續只需提供 GitHub 連結給 Agent，即可自動生成官網。

---

## Step 1: 訪談資料收集

### 訪談時必問項目

| 類別 | 問題 | 備註 |
|------|------|------|
| **基本資訊** | 品牌名稱？公司全名？ | 用於頁尾、版權聲明 |
| **品牌定位** | 一句話形容你們是做什麼的？ | 用於首頁標語 |
| **目標客群** | 主要客戶是誰？ | 影響網站風格和用語 |
| **核心服務/產品** | 有哪些主要服務或產品？ | 用於服務頁面 |
| **品牌故事** | 為什麼創立？有什麼理念？ | 用於關於我們 |
| **聯絡方式** | 電話、地址、Email、營業時間？ | 用於聯絡頁面 |
| **社群連結** | FB、IG、LINE 官方帳號？ | 用於頁尾連結 |
| **特色/優勢** | 跟競爭對手比，你們有什麼不同？ | 用於特色區塊 |
| **常見問題** | 客戶最常問什麼？ | 用於 FAQ 區塊 |
| **行動呼籲** | 希望訪客做什麼？（預約、購買、聯絡） | 用於 CTA 按鈕 |

### 訪談後收集的素材

- [ ] Logo 檔案（PNG 透明背景優先）

- [ ] 產品/服務照片

## Step 4: GitHub 資料夾結構

每個專案在 GitHub 素材庫中應有以下結構：

```
joey-agent-assets/
└── {專案名稱}/
    ├── content.md          # 結構化內容（必要）
    ├── README.md           # 專案說明（可選）
    ├── images/             # 圖片素材
    │   ├── logo.png
    │   ├── hero.jpg
    │   ├── product-1.jpg
    │   └── ...
    ├── documents/          # 文件資料（可選）
    │   ├── menu.pdf
    │   └── catalog.pdf
    └── references/         # 參考資料（可選）
        ├── transcript.txt  # 原始逐字稿
        └── vi-guide.pdf    # VI 手冊
```

### 命名規則

- **專案名稱**：使用英文小寫，用 `-` 連接，如 `hungda-metal`、`taipei-food`
- **圖片檔名**：描述性命名，如 `product-screw-m10.jpg`、`factory-cnc.jpg`
- **避免**：中文檔名、空格、特殊符號

## Step 5: 上傳 GitHub

### 方法一：透過 GitHub 網頁介面

1. 前往 https://github.com/JoeyVIP/joey-agent-assets
2. 點擊 `Add file` → `Create new file`
3. 輸入 `{專案名稱}/content.md`
4. 貼上內容
5. 點擊 `Commit changes`
6. 重複上傳圖片（`Add file` → `Upload files`）

### 方法二：透過 Git 指令

```bash
# Clone 素材庫（第一次）
gh repo clone JoeyVIP/joey-agent-assets
cd joey-agent-assets

# 建立專案資料夾（以宏達精密為例）
mkdir -p hungda-metal/images

# 複製檔案進去
cp /path/to/content.md hungda-metal/
cp /path/to/images/* hungda-metal/images/

# 上傳
git add .
git commit -m "feat: 新增宏達精密專案素材"
git push
```

## Step 6: 啟動 Agent 建站

資料準備完成後，只需提供 GitHub 連結給 Agent：

```
請幫我建立官網
素材連結：https://github.com/JoeyVIP/joey-agent-assets/tree/main/{專案名稱}
```

**範例**（以虛構的宏達精密為例）：
```
請幫我建立宏達精密的官網
素材連結：https://github.com/JoeyVIP/joey-agent-assets/tree/main/hungda-metal
```

Agent 會自動：
1. 下載 GitHub 上的所有素材
2. 讀取 `content.md` 取得網站內容
3. 使用 Next.js + Tailwind CSS 建立網站
4. 部署到 Render

## 常見問題

### Q: 逐字稿很亂，AI 處理後內容不完整怎麼辦？

手動補充缺少的資訊，特別是聯絡方式和產品價格這類必要資訊。

### Q: 沒有 Logo 怎麼辦？

可以先用文字 Logo，或請客戶後續提供再更新。

### Q: 客戶臨時要改內容？

直接在 GitHub 上修改 `content.md`，然後請 Agent 重新建站。

### Q: 圖片太大怎麼辦？

建議壓縮到 500KB 以下，可使用 [TinyPNG](https://tinypng.com/) 或 [Squoosh](https://squoosh.app/)。
